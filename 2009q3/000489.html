Am Sonntag, den 19.07.2009, 13:06 +0300 schrieb Eero Tamminen:
> Btw. Would there be any chance of / would it make any sense to have
> an option for a "dummy" STE Blitter similarly to DSP?  (As I don't see how
> we could optimize the interrupt handling more than by couple of % while
> making the code less readable without somehow minimizing the number
> of interrupts that need to be handled)
> 
> 
> 	- Eero

I think this does not make much sense. An STE with Blitter disabled is
no STE anymore. Better make the emulation code more speedier.

If you disable the Blitter you can really switch the whole emulated
machine over to STF mode.

I think it's more useful to improve the code instead of providing a
senseless disable function.

Also the timing of the Blitter emulation is not accurate according to
the Paradox boys (and they are heavy Blitter abusers on their STE
productions).
