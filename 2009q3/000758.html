> Warning: Soundbuffer size is too big!

Yeah, that happens sometimes when you use a sound frequency bigger than
44.1 kHz. The MIXBUFFER_SIZE = 8192 was chosen when 44.1 kHz was the
highest frequency that the user could select. It's maybe a little bit
too small for higher frequencies.

> In sound.h, I've tryed the following change:
> - #define MIXBUFFER_SIZE    8192          /* Size of circular buffer
> to store sample to (44Khz) */
> + #define MIXBUFFER_SIZE    16384      /* Size of circular buffer to 
> store sample to (44Khz) */
> 
> And the warning doesn't appear anymore. I listen the sound at 50 Khz.
> I haven't uploaded this as the 8192 value is perhaps used for
> something else in Hatari.

16384 should be fine, too. I think it is save to change this value, but
it also shouldn't be much higher because the sound might get more
delayed otherwise.
