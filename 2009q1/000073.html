Hello

my hatari configuration is by default set to STF.

When I sometimes want to debug an STE problem, I turn hatari to STE, run 
the program and do a snapshot that I can reload later.

Now, when reloading the STE snapshot (system is STF), I'm prompted to 
reset hatari for the changes to take effect. This is annoying, because 
this means you first load the snapshot, then reset (so you're no more at 
the point you wanted to restore in the snapshot), then reload the snapshot 
again which is accepted since hatari was reset to STE.

When only the machine type is changed, could there be a way to avoid to 
reset the ST to take the change into account ? I have the feeling that in 
the emulator code, going from STF to STE should not require a reset from 
user (since most "Reset" functions are called when restoring).

Perhaps other parameters in a snapshot should require a reset when they 
change, but in that case, perhaps we should have 2 messages :
  - one that say a reset is needed, and we can choose ok or cancel (same as
    today)
  - one that just say "snapshot reloaded" (and perhaps machine type
    changed), and you just press OK.

Eventually, I think a snapshot should be reloadable and restored to the 
exact point where is was saved, without requiring the user one extra step 
to reset.

What do you think ?


Nicolas
