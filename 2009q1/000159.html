Hi,

While testing new DSP code with couple of programs, I noticed a problem that
happens only with Falcon/TT mode...  If program sets 8-bit mode, all colors
to same value and freezes, what user is to do?  He doesn't see the options
dialog or quit confirmation dialog (white on white...).  It just seems like
Hatari itself had frozen.  I had to use Gdb to notice that Hatari actually
hadn't frozen.

Maybe Hatari in 8-bit mode should save & set palette when entering options
or other dialogs and restore the palette when leaving them?

Other option would be to use 16/24/32 SDL screen even when the emulated
system uses 8-bits.  It adds a bit of overhead, but the conversion isn't
*that* heavy, at least when using frameskip (this way statusbar colors would
also be OK :-)).
