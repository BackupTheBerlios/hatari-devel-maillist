Hi,

On Thursday 19 February 2009, Matthias Arndt wrote:
> Allright, I did a small patch which addresses the needs of some modern
> gamers. They prefer to press a second firebutton on their gamepads to
> jump in Jump'n'Run games so I added this to Hatari in an optional way:
>
> I added a configuration option (lacks GUI!) bEnableJumpOnFire2 for each
> joystick. If this is TRUE, pressing PC joystick button 2 will now not
> press SPACE anymore but emulate pushing the ST joystick UP which maps to
> jumping in most jumper games.
>
> Except for making this configurabl in the GUI, it should work. I tried
> with Giana Sisters and my USB gamepad. By default the option is set to
> FALSE to have the old default behaviour of pressing SPACE.
>
> Comments and code reviews welcome!

The cosmetic changes were fine, but new features & options should be
discussed on the devel mailing list first.

A single hack like JoystickSpaceBar is quite OK, additional ones aren't
anymore.  New ones indicate that there should be some generic mechanism
for setting what the different joysticks buttons do (generate joy events,
key presses...).  Could you come up with a proposal for that?

Then you would need to add a command line option(s) for this and update
the manual page.  GUI configuration would be nice too. That requires
updating the HTML manual.
