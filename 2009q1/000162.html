Hi,

On Friday 13 February 2009, Jerome Vernet wrote:
> > While testing new DSP code with couple of programs, I noticed a problem
> > that happens only with Falcon/TT mode...  If program sets 8-bit mode,
> > all colors to same value and freezes, what user is to do?  He doesn't
> > see the options dialog or quit confirmation dialog (white on white...).
> >  It just seems like Hatari itself had frozen.  I had to use Gdb to
> > notice that Hatari actually hadn't frozen.
> >
> > Maybe Hatari in 8-bit mode should save & set palette when entering
> > options or other dialogs and restore the palette when leaving them?
>
> The same way, I noticed that the Status bar sometime change, when the
> palette change. It's happen, I don't know why, with the Cecile driver
> with MagiC.

This is because 8-bit mode has palette and Statusbar needs to use colors.
It can easily happen that an emulated (Falcon) program changes a palette
entry used by Statusbar.

An alternative would be on each frame to re-find a suitable color and redraw
the whole statusbar, but as this is a problem only on 8-bit mode, I didn't
think it worth slowing down the screen updates.
