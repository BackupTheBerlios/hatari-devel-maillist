Hm.  While reading about Gnome Svn -> Git transition, I noticed a comment
about push vs. fetch.  Hg has fetch command too:
--------------
$ hg help fetch
hg fetch [SOURCE]

Pull changes from a remote repository, merge new changes if needed.

    This finds all changes from the repository at the specified path
    or URL and adds them to the local repository.

    If the pulled changes add a new head, the head is automatically
    merged, and the result of the merge is committed.  Otherwise, the
    working directory is updated to include the new changes.

    When a merge occurs, the newly pulled changes are assumed to be
    "authoritative".  The head of the new changes is used as the first
    parent, with local changes as the second.  To switch the merge
    order, use --switch-parent.
--------------

Maybe this doesn't move the repo head between my and Thomas' hg pushes, like
using pull+merge does.  Thomas, do you know?


My Hg is missing[1] "rebase"[2] command that Git has, so there's still the
extra merge commit.

[1] Well, at the end of last year HG got an extension for this, but I'm
running an older Hg version.

[2] rebase = updating your current repo to same state as remote repo after
fetch/pull instead of needing merge+commit.
