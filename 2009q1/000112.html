I just wanted to have a spot check with people that the way i'm creating my diffs is the best for everyone.
 
I've used:
 
hg diff -u 5
 
(from my local repositry) this though seems not to have the -N option. So for my current update (VC compatibility) i'm using:
 
diff hatari_base hatari -E -b -w -B -p -r -N  -d -U 5 > difffile.txt
Where 'hatari_base' is the original clone before updates where made and 'hatari' is the modified source.
 
----
 
I am thinking, that I would use the first syntax when I have no additional files and the second for the situation where I need to add files.
 
Hope thats right :).
 
Looking forward / keen to get the code VC compatible so I can get to work on the more exiting GEM DOS HDD stuff.
