As these functions just access specific memory addresses and don't really
depend on the other Videl or Crossbar code, I think it would be nicer to
have them directly in debug/info.c (along with anything else you think
useful).  That way e.g. the automated debugger test code (in tests/ dir)
doesn't need to link stuff that rest of videl.o or crossbar.o depend on.


	- Eero

PS. I personally prefer "early exit" coding style:
	if (ConfigureParams.System.nMachineType != MACHINE_FALCON) {
		fprintf(stderr, "No Videl\n");
		return;
	}
	/* actual functionality */

Instead of nested ifs, but that doesn't really matter. :-)
