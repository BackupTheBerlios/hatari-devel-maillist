On Sunday 21 March 2010, Eero Tamminen wrote:
> I added code to check largest available resolution and to limit the user
> provided maximum resolution to that.  It's checked on each screen size
> change as it depends on the fullscreen flag.

After this change, I see following uses for the "Max zoomed" option:

* Main one is providing goal for Falcon/TT screen size, especially in
  windowed mode as SDL & OS allow windows to be of arbitrary size and
  because Falcon resolution sizes can vary wildly.

* Using that as guideline on whether to double ST/STE low rez and limit on
   how much of the borders should be used.  Latter helps user in better
   matching Atari screen size to a suitable fullscreen resolution than the
  earlier BorderPixel values that could be changed only from the Hatari
  configuration file.

* Making Hatari use smaller zooming/resolution on fullscreen than
   the default.  This is mainly to speed up the screen updates slightly
   by letting the monitor to do the zooming instead of Hatari.

Currently the same maximum size is used both for windowed and fullscreen
mode.  Do you think they should be separately configurable?


(I'm also open for using different name for the option, if you have good
arguments why something else would be better.)
