Hi,

On Friday 29 January 2010, Laurent Sallafranque wrote:
> While testing the latest GemPlay (seen on DHS site), I've got the
> following message :
> (I'm trying to open a folder from the file selector).
>
> GEMDOS dir name '*TUNES' with wildcards in *TUNES\*.*!
> No GEMDOS dir
> '/media/Jeux/Jeux/Atari/DiskDur/Utils/Falcon/GEMPLAY/*TUNES'

It's a bug in latest GemPlay (documented for quite long time already
in compatibility list), it works only for files in the drive root.

Atari GEMDOS supports wildcards only in Fsfirst() call, and only for
the filename part:
"This may use wildcard characters (? or *) within the filename, however they 
may not be used within the pathname. This function is the only GEMDOS 
function which accepts wildcard characters in the path specification."

I.e. they're illegal (or literal?) in directory names.
