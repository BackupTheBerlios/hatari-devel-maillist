On Thursday 14 January 2010, Eero Tamminen wrote:
> It can now  be used from debugger (e.g. "info videl"), with TAB
> completion.

It has now "osheader" and "basepage" subcommands to find out some OS address
information (that could be interesting to memdump) and to find out into
which offset to load the currently running program symbols.

It's now also possible to give an optional (Uint32) argument for the info
subcommands (currently used only for the basepage command).
