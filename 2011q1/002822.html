Hi,

I've uploaded a first version of new Hatari with winUae core and DSP.

I've tested it a lot.
I haven't applied the patchs yet, I wanted to test DSP as closely as it 
is actually with old cpu core.

I've tested several demos and progs.
I haven't noticed any regression.

Eko system still needs 32 Mhz to work.


Better, some programs are now working :
K.prg
Yepiha goes a bit far than before
Sonolite demo seems better too (the Avena logo didn't appear before)

Sound seems cleaner too (not perfect with every demo, but much cleaner 
in general).

Don't hesitate to do some tests.

My "config" : 68030, 16 Mhz, tos 4.04 14 Mg ram, DSP full.
CPU parameters : 24 bit adressing, no prefetch mode (for now), no 
cycle_exact (this option crash for now), Real time clock emulation, no 
patch timerD, Blitter emulation, no MMU.

for FPU  : CPU internal, More compatible but slower.


I continue to "play" with the new CPU.
