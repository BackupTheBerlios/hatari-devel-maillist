As can be seen from the Hatari help heading under which this option is,
it affects only Falcon/TT display.

This is because the Videl emulation code used for their screens supports
arbitrary integer scaling factors, whereas the screen handling functionality
for ST & STE screens doesn't, it supports just doubling[1].

Anyway, that should be less of an issue for ST & STE emulation as they
aren't constantly changing resolutions like (some) Falcon demos...

[1] ST & STE screen handling functions on the other hand have the advantage
that they use much less CPU if nothing changes on the screen.  This is more
important on machines (handhelds, phones etc) that have barely enough power
to emulate ST/STE.
