That's on purpose as they (mainly) apply only to Falcon/TT.

"Max zoomed" enables the ST low-rez doubling if the selected resolution
is large enough, but that's all the effect it has for that.  For VDI mode
it doesn't have any effect.  It's already explained in the manual.


The problem is that Hatari has 3 different video output systems,
ST/STE, VDI and Videl which is used for Falcon & TT.  If the Videl code
would be doing updates only if something changes (currently it always
updates everything which makes it much slower for GEM apps), then it
would be IMHO best to move at least the VDI code to that too...


> Maybe you could split this screen in several horizontal parts :

> - a Falcon/TT part with "keep desktop res".

> In case we don't have enough room, maybe we could move the "screen
