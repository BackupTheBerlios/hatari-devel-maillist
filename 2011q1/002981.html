The videl have a bug regarding updates of the colour registers. When 
updating a colour register, the videl will show garbage pixels meanwhile 
the colour is set. This makes it impossible to implement classic ST 
plasmas or Spectrum 512 things. As if the variable bus load and caches 
weren't enough to put you off already :-)

Other games which work on Falcon that uses some STe video features are 
Stardust (top/bottom borders), Obsession (top/bottom/left borders as well 
as screen splits), Utopos (top/bottom borders and screen splits), 
Substation (top/bottom borders in the intro).

I think in Obsession on Falcon/VGA you can see the videl bug with garbage 
pixels during colour change. On VGA the HBL moves twice as fast and there 
are no borders, so the colour switch comes in the middle of the screen 
there hence showing the bug.
