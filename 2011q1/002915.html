I wouldn't suggest using inaccurate emulation, at least if you're doing
anything related to demos. :-)

Besides timer-D, disable DSP emulation by default and enable it only
when you need it.

If you're using compositing or are running your desktop at 32 bit depth, you
could try switching compositing off and running the display at 16-bits[1].
If whatever you're running doesn't do lots of resolution switching, I would
also suggest setting the Hatari max zoomed size low and using full screen.
Those should lessen the screen conversion overhead which is important if you
want to run with 0-1 frame skips.  


	- Eero

[1] AFAIK many LCD monitors don't show 32-bits of colors, the color
support in really cheap ones could be even as low as 18-bit.
