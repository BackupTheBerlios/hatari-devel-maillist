OK, found : it's because of the patch I added in stMemory.c

This mean that all the descriptions of the $ff8006.b register I read 
were wrong.

Anybody here as a complete description of this register ?

For monitor :

		/* Set the Falcon memory and monitor configuration register:
			$FFFF8006: Monitor Type BIT 7 6  (These 2 bits are duplicated in $FFFF82C0 (BIT 1 0))
				00 - Monochrome (SM124)
				01 - Color (SC1224)
				10 - VGA Color
				11 - Television

For memory :
			  $FFFF8006: Memory configuration BIT 5 4
				00 -  1Mb
				01 -  4Mb
				10 - 14Mb
				11 - no boot !


To see the memory of my Falcon, I just wait the memory test at boot to 
complete.
It displays memory size.


If I write $20 to the register, I get only 8 meg (???) instead of 14.
(8 Meg is possible on Falcon ? I thought it was 1, 4 or 14 megs only)

If I keep the original value $26, I've got 14 megs as expected.

So what is the bits 3, 2, 1 and 0 meaning ?
