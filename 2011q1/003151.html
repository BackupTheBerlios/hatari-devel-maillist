Hi,

I've now tested the new --desktop option on following setup:
- Ubuntu v10.4 (Lucid)
- XFCE v4.6.1
- SDL v1.2.14
- Linux v2.6.32
- X server core v1.7.6
- ATI Radeon RV100 card
- X ATI Radeon driver v6.13.0
- Samsung CRT-monitor

On this setup --desktop option (requested by Anders) works fine
and makes fullscreen much nicer with LCD screens.

(Because Videl code supports only integer scaling factors, it works
better if you have a large host Desktop resolution, for smaller host
resolutions some larger Atari resolutions don't get scaled because they are
too large for 2x scaling and this can result in large variation of visible
screen sizes.)


On my setup:
- Debian Squeeze
- KDE v4.4.5
- SDL v1.2.14
- Linux v2.6.32
- X server core v1.7.7
- Intel HD graphics integrated into i3 CPU
- X Intel driver v2.13.0
- HP LCD-monitor

I sometimes get a totally black screen on toggling fullscreen, just
the mouse shows.  Screen remains like that even after I kill Hatari.

It seems to be this bug in X server Intel driver:
	http://lists.debian.org/debian-x/2010/11/msg00125.html
	https://bugs.freedesktop.org/show_bug.cgi?id=31367

Which is supposed to be fixed two weeks ago in Debian, but I still don't
see the fix in Squeeze (version -6 instead -5 of the Intel driver). :-/


I would be interested to hear how well this option works on OSX or Windows.


	- Eero
