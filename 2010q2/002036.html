Hi,

On Monday 10 May 2010, npomarede at corp.free.fr wrote:
> I agree that if the user choose a TOS version and we can identify this
> TOS and its cpu/blitter/video/whatever requirement, then it would be a
> good idea to force all the parameters (not only cpu type).
>
> But do we have such a list ? Some tos versions can be compatible with
> several cpu/speed, in that case we should have a default setting.

I think all TOSes work with all CPU speeds Hatari supports.  However,
programs like overscan demos, are picky about CPU speed and don't
work with "random" CPU speeds.

If user doesn't have the statusbar enabled, Hatari claiming to switch to
Falcon mode, but leaving CPU speed e.g. at 8Mhz could be quite confusing
("I didn't remember this demo being this slow" etc).


> Also, this should not prevent the user to specify "exotic" combinations
> when using the command line, for example "--tos tos_tt --cpuclock 16"
> should still be possible ; in that case we would read arguments from left
> to right, the rightmost argument would override the 32 Mhz implied by
> "--tos tos_tt"

At argument parsing time, the last given argument "wins".  TOS version
vs. CPU & machine type checks happen only after Hatari starts booting.

As long as user gives a working configuration, exotic combinations work
just fine.
