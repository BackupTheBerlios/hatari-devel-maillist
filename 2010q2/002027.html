I think easiest would be to make the save & load config buttons to

Hatari (python/Gtk) UI already does this.  Just use the "load config..." and
"save config as..." options from its Configuration menu.


> Like this, I would easily choose between a ste and a Falcon at boot and
> I could customize my virtual computers once for all.

With current Hatari you can already do this, just:
- save a suitable configuration & memory states to disk
- copy them under suitable name (say falcon.sav & falcon.cfg)
- set config file to load that memory state automatically (& autosave it)
- create to your desktop shortcut icons which tell Hatari to load these
  configurations.  E.g. "Falcon" shortcut would just start Hatari like this:
	hatari --config <path>/falcon.cfg

The SDL GUI is not really that fun to modify.  Position of everything are /
currently need to be hard coded which means that if you add something, you
need to change also everything else below it (and maybe adapt sizes of
surrounding boxes too).

Additionally, it doesn't handle non-ASCII file names, isn't localizable,
doesn't scale with GUI contents etc.  These are some of the reasons why
I wrote the Hatari UI and why it's included with Hatari...
