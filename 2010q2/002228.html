Hello,

 From my point of view, the actual code is correct enough for the new 
release (I speak for the code I modified).

DSP emulation let run more new programs and musics.
STe DMA sound is really much better (LCM and microWire emulation)
Many bugs fixed.
Info crossbar added


For after the 1.4 release :

- I would like to add all the DSP traces (cpu, host transfers, SSI 
transfers, interrupts)

- I've noticed a few regressions with some Falcon demos (Eko), AFM 
doesn't work again (Winrec still work).

- Sound is still not good sometimes (I've got an idea I'd like to test : 
it deals with SSI transfers when crossbar plays 8 bit sound but SSI is 
configured for 16 bits transfers)

- PowerUp Ste still doesn't sound good (I'll need Nicolas' help when 
he's back for this one)

- There are still some programs that run only when CPU is set to 32 Mhz

- There's the APEX $fff20000 strange address problem (Thomas, do you 
think we should hack this problem with the patch you sent me ? (we would 
have to add a lot of comments to say this is only a bad hack for now))

- And the 2 biggest problems to solve :
     o Videl emulation has to be full rewritten
     o Main CPU should be upgraded from WinUae CPU (MMU + bus cycles)

But all these points should be taken into account after the 1.4 release

Last info, I'll probably have a Falcon this summer. YESSSSS :)
I'll be able to tests things directly on the real hardware, and I won't 
have to guess everything all the time.

So, go for the 1.4 release :)

Best regard to you all,

Laurent
