> > > set(CMAKE_REQUIRED_LIBRARIES "mudflap")
> > > CHECK_C_COMPILER_FLAG("-fmudflapth" MUDFLAP_AVAILABLE)
> > > set(CMAKE_REQUIRED_LIBRARIES "")
> 
> I noticed that later in the file readline check does something
> similar: ------------
> if(READLINE_FOUND)
>         set(CMAKE_REQUIRED_LIBRARIES "readline")
>         check_function_exists(rl_filename_completion_function
> HAVE_RL_... if(HAVE_RL_COMPLETION_FUNCTION)
>                 set(HAVE_LIBREADLINE 1)
>         else()
>                 unset(READLINE_FOUND)
>         endif(HAVE_RL_COMPLETION_FUNCTION)
>         set(CMAKE_REQUIRED_LIBRARIES "")
> endif(READLINE_FOUND)
> ------------
> 
> But shouldn't the CMAKE_REQUIRED_LIBRARIES be restored to the previous
> value instead of an empty one?  Or is it used only for doing tests
> like this, not building the software[1]?
> 
> (CMake manual doesn't tell.)

CMAKE_REQUIRED_LIBRARIES is just for the configuration tests, for the
final linking the target_link_libraries() is used, e.g. see the file
src/CMakeLists.txt in the Hatari repository. 


> Building with mudflap is now:
> 	cd build; make clean; cmake -D ENABLE_MUDFLAP:BOOL=1 ..
> instead of:
> 	make clean; make MUDFLAP=1
> 
> (documented in the main CMakeLists.txt.)

Ok, good to know, thanks!
