The remote console thing is mainly intended to (eventually) automate
things like a test-suite for Hatari; running things inside and outside
Hatari with python scripting, but I haven't yet had time to test that
properly.

configuration match it, that's why it starts fine.

Hatari UI runs "hatari" from your PATH.  If you haven't installed latest
Hatari to your system (which I assume to be the issue), use something like:
	cd python-ui
	PATH=../src:$PATH ./hatariui
instead.

the console?
