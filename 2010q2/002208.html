> remove only ";ppc". If not, under 10.6.3, gcc will try to build a
> x86_64 (by default) app, wich will cause error with SDL 1.2 (wich is
> NOT 64 bits on MacOsX and use deprecated interfaces with Cocoa
> 64Bits).

I think it would be better to fix the "poseAsClass" problem in
SDLMain.m instead of forcing a 32-bit build... The SDLMain.m from the
latest SDL 1.2 has this problem fixed, so someone who is still able to
build the OS X code should port this fix to our SDLMain.m, too.

Actually, it seems like Hatari now does not run properly on Mac OS X
10.4 anymore - your new .nib files really seem only to work with 10.5
upwards (I just checked it again with your latest update, Jerome).

That means I (and most likely most other PowerPC users) can not use
Hatari on OS X anymore, so I simply removed the ";ppc" from the
CMakeLists.txt file now ==> Support for Hatari on PowerMacs / Mac OS X
older than 10.5 is now over, but at least the build problems with newer
OS X versions should be fixed now.
