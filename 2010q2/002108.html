Anyway, I came over this, causing my ugly hardcoded 32 bit GL "blit"
to crash (yes i will clean it up before submitting anything);

in PrefsController.m at line 787 (atleast before committing Jeromes
patches), there is this line:

	ConfigureParams.Screen.nForceBpp = ([force8bpp state] == NSOnState) ? 8 : 16;

This means, whenever you touch the OS X GUI and bpp != 8 (NEVER
happens) you get nforceBpp = 16 in the config file. Ouch. So, not only
is performance already troubled by the lousy SDL 1.2 implementation on
OS X - hatari OS X GUI is making it go an extra round and convert it
up to 32bpp... This makes no sense.
