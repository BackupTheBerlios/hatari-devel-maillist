Hi,

There are a couple of asserts in Hatari.
I think they should be fixed for the release.

You can get them like this:

1) Run the EPSS STE music demo.

2) Start Hatari with e.g. TOS 1.4 and something like this:
	./hatari -m -d includes/ --mount-changes no
  Open drive C: and drag a file within the window so that the GEM
  data copy dialog opens.  Enable fast-forward.

Then wait and within about 5 minutes Hatari should in both cases die to
this assert:
hatari: cycInt.c:377: CycInt_AddRelativeInterruptWithOffset: Assertion 
`CycleTime >= 0' failed.

First one seems to be 100% reproducible, second one may need longer.
When I earlier debugged the monochrome one, it was video related, I think
the EPSS demo one is also.

Nicolas, could you check whether you can reproduce these
(at least EPSS one) and debug them?
