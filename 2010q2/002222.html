Hi,

I rewrote hmsa tool to support also empty disk creation in addition to
converting between disk formats as I plan to use both of these features
from the Python UI later.

At the same time I made it a bit more robust against user errors by refusing
to overwrite an already existing file and added a manual page for it.
Readme is an ASCII version of the manual page and there's a rule in Makefile
to re-create it.


What do you think, would it be OK to add this before v1.4 release?

The actual ST & MSA disk creation is done by already existing Hatari
code, hmsa itself is just a wrapper for these so testing its own
functionality is trivial (seems to work fine).

Most annoying thing is CMakeLists.txt, I copied the manual page kludge
from doc/CMakeLists.txt, but it's ugly to duplicate code for creating
manpages, isn't it possible to share the CMake code for that?
