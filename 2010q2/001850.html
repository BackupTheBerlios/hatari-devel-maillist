On Fri, 16 Apr 2010, Christer Solskogen wrote:

> The new configure system supports cross-compiling
> (--cross-compile-win32) to Windows. Have anyone tried it? If you have
> or do use it, could you post how you created your "cross world" with
> libsdl, portaudio and png?


Hello,

I tried it, since I added this option myself :)

Depending on your linux distribution, you should have some "mingw" 
packages to install. In my case (under Mandriva), I installed 
mingw32-binutils, mingw32-gcc, mingw32-libpng, mingw32-readline, 
mingw32-w32api.

Rule of thumb is  : run "--cross-compile-win32" and each time an error is 
reported, look for a corresponding package for mingw in your distrib.

In my case, sdl was not available, so I got the version from official 
mingw site. Just installing this version (specifying the correct 
destination path, so it goes in the usual cross compile path defined by 
your mingw32-gcc package) was enough to cross compile and get a windows 
binary.


Nicolas
