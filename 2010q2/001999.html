On Thu, 6 May 2010, Eero Tamminen wrote:
>> On OS X again, if you set frameskip to auto, you get a 5x ffwd (max
>> frameskip). Or so it appears.
>
> Max. frameskip can also be configured from the Hatari configuration file.
>
> The current default 5 frames max skip is selected as maximum still barely
> usable value when NOT using fast-forward (AltGr-X shortcut).
>
> -> Larger frameskip values don't help with performance, if the host computer
> is so slow that it doesn't keep up with Hatari even when using frameskip.
>
> (One could be running Hatari on a mobile device, or using Falcon+DSP
> emulation on something like my own 7 years old computer.)

It's true that frameskip can be useful for slower pc/device, but I must 
admit that when I use steem to test some things, I really enjoy the fast 
forward feature, because it really goes fast :)  Much faster than Hatari 
alt+x fast forward mode (using nFrameSkips=0 in my case, because I don't 
want frame to be dropped in the normal case)

Maybe we could keep the fast forward key in Hatari and add a "super fast 
forward" key that would temporarily increase max frameskip to get an 
effect similar to steem (without having to edit config file to temporarily 
change some params and restore them later) ?
