All Hatari keyboard shortcuts are configurable from its configuration file.

Attached is source for a program which shows on console the SDL keycode for
the key you press.  You can use it to get a keycode you need to put to
the Hatari configuration file.

The program is from the SDL sources and can (on Linux) compiled like this:
	gcc -O -o checkkeys $(pkg-config --cflags sdl --libs sdl) checkkeys.c

Would it make sense to add this to Hatari sources tests/ directory too?

I could maybe at some later state add support for configuring Hatari 
keyboard shortcuts to the Hatari (PyGtk) UI.


I attached also a trivial Atari program for seeing what Atari keycodes you
get when pressing some key in Hatari. :-)


> > On OS X it maximses to VBL speed in window-mode (so practicly going to
> > 60Hz instead of 50Hz.. just 20% speedup).

This is done by OSX version of SDL I think, it definitely isn't done
by Hatari.  If it's done by SDL, maybe there's some environment
variable with which it could be disabled?


> On OS X again, if you set frameskip to auto, you get a 5x ffwd (max
> frameskip). Or so it appears.

Max. frameskip can also be configured from the Hatari configuration file.

The current default 5 frames max skip is selected as maximum still barely
usable value when NOT using fast-forward (AltGr-X shortcut).

-> Larger frameskip values don't help with performance, if the host computer
is so slow that it doesn't keep up with Hatari even when using frameskip.

(One could be running Hatari on a mobile device, or using Falcon+DSP 
emulation on something like my own 7 years old computer.)
