> Hatari default mouse behaviour (ungrabbed and unsynced) is very annoying.

Grabbing a mouse is quite user unfriendly when (a new) user doesn't
know how to disable it or even exit Hatari...  If this is an issue, I'd
recommend running Hatari in fullscreen unless you need the debugger.

We could consider adding a config file option for mouse grabbing so that you
don't need to specifically toggle it everytime with keyboard or command line
option.  (Then the command line option needs to take a boolean argument
so that grabbing can be turned off from the command line.)

As to unsynched mouse cursor, I think that's mainly(?) a problem with
Falcon, and I don't think Steem is any better there (the issue comes from
Falcon's multiple resolution changes at bootup). :-)


> Steem has very intuitive memory snapshots accessible from the toolbar.

Hatari UI allows selecting the memory snapshot with a normal Gtk
file selector for loading, there's a menu entry for saving one (to default
location) and for changing the snapshot default location.
