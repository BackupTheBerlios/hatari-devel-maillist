Hi,

In case you're interested I just tried a bit newer version of MiNT.

I got Micro's uMiNT package (for >=040 machines) from here:
	http://mikro.umpc.sk/download.htm

And turned its contents into a HD image with the Hatari atari-hd-image.sh 
script:
	$ unzip uMiNT.zip
	$ atari-hd-image.sh 64 umint.img MINT uMiNT

The booting seems a bit complicated.  For some reason the auto/-folder
stuff isn't executed from the image, so I need to invoke the early boot
console from EmuTOS, "cd auto/" and run mint117.prg manually (memory
protection needs to be turned off).  Then it boots fine:
$ hatari --tos etos512k.img --machine falcon --cpulevel 4 --cpuclock 32 -s 
14 --dsp none --acsi umint.img

Screenshot is attached.

Any idea why auto/-stuff isn't run with EmuTOS?

When using GEMDOS emulation directly for the uMiNT directory, the stuff
boots fine from auto/, but naturally doesn't work otherwise (because MiNT
doesn't recognice C: once it has booted up).

If I use Cecile bootdisk (and --ide) and put MiNT stuff to the same floppy
image, it will bomb at boot.


Some minor issues with uMiNT:
* At least with EmuTOS, the screen resolution changing from XaAES or
  Teradesk doesn't work.
* Some of the desktop icons have wrong paths (use d:\mint\ instead of c:\).


	- Eero

PS. Does somebody have binaries for fVDI & drivers & native setup for it?
The packages I found were either old or for Aranym NatFeat stuff.
