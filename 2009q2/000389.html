Hi,

On Tuesday 16 June 2009, Laurent Sallafranque wrote:
> If I use a "dd 0 100" command to disassemble code from p:0 to p:100, the
> debugger only displays 8 instructions.
>
> In range mode, would it be possible to have the whole range displayed at
> once ?

It does. The range syntax is <start>-<end>.  You need to use "dd 0-100".

DebugUI could warn if one gives extra arguments though.

Thomas, in breakcond.c I pass a pointer to a structure that contains
the information about the arguments.  Each of the called functions updates
the information in this struct about the argument to be processed next.
The topmost code warns if there are unprocessed args.  Maybe something
similar could be used in debugui.c also?
