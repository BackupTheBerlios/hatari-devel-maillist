Needs to be configurable. I do like the idea myself, but I can also see your point Thomas that if its not what your needing it might just disturb other information already displayed.

 

This brings me to a nice conclusion (for myself). What we could do with is an HAtari API for third party applications. One of the "plug in" points could be "on BUS error" or OnDebug. 

 

If I was doing this for a Windows project I would have a plug-in directory from which the application would load any DLL's in the directory. The DLL's would provide Exports which HAtari would understand (like HAtari_OnBUSError and HAtari_OnConfigChange or whatever) and call at the appropriate times. 

 

If we provided an API like that for HAtari then people could more easily make little customisations for themselves (and provided them for others). Thats as far as i've thought so far. What thoughts everyone?
