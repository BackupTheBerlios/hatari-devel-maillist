> What I did was:
> ---------------------
> Get Hatari CVS repo from Sourceforge to a work dir:
>     mkdir repo; cd repo
>     rsync -av hatari.cvs.sourceforge.net::cvsroot/hatari/*
> 
> Point CVS to workdir:
>     export CVSROOT=/home/eero/repo
> 
> Checkout python-ui module:
>     mkdir checkout; cd checkout
>     cvs checkout python-ui
> 
> Convert it and its history to Mercurial repo:
>     hg convert python-ui
> 
> Mergeing that to main Hatari repo:
>     hg clone hatari-hg work         # pull from both repos...
>     cd work
>     hg pull -f ../python-ui-hg
>     hg update -C tip                # Now update to python-ui-hg
> tip... mkdir python-ui                 # And put things to subdir by 
> renaming...
>     hg rename <dirs & files> python-ui
>     hg commit -m "Move everything into python-ui directory."
> ---------------------
> 
> Could I have done at my end what you did?  How?

Since the files should be put into the python-ui folder, it was a
little bit difficult: I first exported your patches with hg export :

x=1851 ; while [ $x -lt 1968 ]; do \
   hg export $x > /tmp/patch-$x.txt ; let x=x+1; \
done

And then imported them again with these commands:

for i in /tmp/patch-1*.txt ; do \
  cat $i | sed 's,+++ b/,+++ b/python-ui/,' \
     | sed 's,--- a/,--- a/python-ui/,' \
     | hg import -u "Eero Tamminen <eerot at users.berlios.de>" -d - \
; done
