> Does it takes care of the 68030 cache ?

I haven't seen this in the code yet, so no, but I might be wrong.

> And does it takes care of the instructions pairing ?

No, instruction pairing is Atari specific. But the WinUAE core features
full sub-cycles emulation during instruction decoding, not only at the
end of each instruction, so this might help to improve the precision of
some emulator effects.
