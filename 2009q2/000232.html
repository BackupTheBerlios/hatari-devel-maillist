Hi,

On Saturday 04 April 2009, npomarede at corp.free.fr wrote:
> What about a flag to tell what to do when we're in debug mode and the
> pause key is pressed ? For example --debug-start = none or dump_reg or
> dump_code or dump_all or ... ?

That should be pretty easy to do.  Maybe just:
  --debug-start <r|d> Show at debugger start: r=registers, d=disassembly.

And if one gives "d" multiple times, it will disassemble more
(call the disassembling function multiple times).

Does this sound useful?


> > Or maybe there should be a separate debug mode that would just dump the
> > info and then continue emulation?  (then -D could take an argument for
> > what it dumps whenever debugger key is pressed)
>
> considering the speed at which the 68000 is changing its state, I don't
> see many interest in dumping the content of the register and continuing
> the emulation. I don't think anyone will succeed in dumping what he
> expects at the exact time.

I was thinking it more of a something one uses to kind of "sample" what
the emulated program is doing.

But being able to set breakpoints into the emulated memory and to be able to
single step the program execution would be much more interesting features.
(You've been mentioning that you could look into the latter... :-))
