> I'd like to add a DSP breakpoint function.

Maybe "db <address>" command?

If you write the function to do the thing on the DSP side, I can add
what's required in the debug UI.

Btw. How you you've thought to implement it?  Adding a single "breakpoint
address register" to the DSP emulation, or using some kind of a breakpoint
instruction which the debugger can use to replace the instruction you're
interested about and which will result in DSP emu calling the debug UI?

Debug UI would then need to restore the instruction, singlestep that
instruction, replace instruction again with the breakpoint instruction and
continue.

Latter would allow multiple breakpoints and e.g. counters (debug UI would
continue running the code until breakpoint has been hit <count> times) , but
would be more complicated (needs also singlestep functionality etc).


> My only problem is : how can I stop hatari and open the debugmode ?

Start Hatari with the -D option and press Pause/Break key.

(Without -D, Pause key will just pause emulation.)
