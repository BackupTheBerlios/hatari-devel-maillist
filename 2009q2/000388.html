On Tue, 16 Jun 2009, Eero Tamminen wrote:

>> (for example, some demos could store packed data at
>> odd address, and then you could do something like (a0) = 'ICE!' to add a
>> breakpoint (hint : we could also have the possibility to use 4 ascii
>> chars as a 32 bits value :-) ))
>
> What kind of a syntax/prefix you'd thought for ASCII and binary values?
>
> Should they be allowed also for specifying addresses, or is it enough
> that addresses can be specified only as decimal or hexadecimal?
>

Well, I don't think it should be used for addresses, I don't see real case 
use for sthg like ('1234') = $12 for example.

But wherever a constant value is expected, I think we could convert :
  - 'x' to a 8 bits value representing ascii value of "x"
  - 'xy' to 16 bits ; value = val(x)<<8 + val(y)
  - 'wxyz' to 32 bits ; value = val(w)<<24 + val(x)<<16 + val(y)<<8 + val(z)

So, '01' = $3031 = 12337

As for binary number, asm convention is to prefix with '%', with only 0 or 
1 after (max of 32 0/1). I don't think number of 0/1 should be enforced to 
be a multiple of 8, we can just consider extra 0 in front can be removed.

So, %1000 = 8 = %000001000


>
> Btw. Decimal is currently the default number format in conditional
> breakpoints, hexadecimal values need to have the '$' prefix as you
> requested earlier.

Yes, I saw that ; adding $ for hexa was also important as it allows to 
make the difference between $a0 (the value) and a0 (the 68000 reg), 
without $ there would be an ambiguity to find the meaning of 'a0'
