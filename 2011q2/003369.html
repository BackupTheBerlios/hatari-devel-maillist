Hello

this demo is now fixed ; it used an address error exception (PC at odd 
address) during an illegal instruction to decode the program, which was 
not correctly emulated (exception processing didn't check the new PC was 
even).

Note that you need to disable (eject) HD emulation in order to run this 
demo, because it uses opcode 0x0008 as an illegal instruction and 0x0008 
is reused by HD emulation and will prevent the correct exception to happen.
