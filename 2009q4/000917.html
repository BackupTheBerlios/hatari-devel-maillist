> On Sunday 15 November 2009, Uwe Seimet wrote:
> > > As the UAE core Hatari uses doesn't emulate MMU, 020 + FPU is
> > > actually the most accurate description for the processor.
> > >
> > > (The 040 option doesn't support MMU either.)
> >
> > Nevertheless an additional option for a 030 even without MMU might
> > increase Hatari's compatibiliy. Some programs might check for both
> > the _CPU and the _MCH cookie and might not work as expected if they
> > find a Falcon with a CPU older than a 030, even if they do not use
> > the MMU. A Falcon with a 030 without MMU is more a real Atari than
> > a Falcon with a 020 without MMU or a 040 without MMU. This is why I
> > opt for Hatari to support an option for setting _CPU to a 030.
> > If I remember correctly there even was a 030 without MMU offered by
> > Motorola: THe 68030EC.
> 
> Fair enough.   Nicolas & Thomas, any comments?

The cookie jar is set by TOS, not by Hatari, so we don't have any
direct influence on that. So the only option would be to improve the
CPU emulation core. Does anybody know what TOS 4.0x is doing to
distinguish an 68030 from a 68020 ?
