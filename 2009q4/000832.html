That's how it works.

However, there are two gotchas with it:

* If the AVI file is small (not hundreds of MBs), the dialog flickers too
  quickly on screen for user to see anything.  Because of second item,
  this can seem a bit strange.

* Because I refactored only AlertDialog stuff to have separate showing
  and hiding functions, but didn't refactor the DoDialog() (in another SDL
  GUI *.c file) into separate background saving and restoring functions,
  once the notice dialog (automatically) closes, it's left still visible
  behind the other dialogs.  All other dialogs can use doDialog() so they
  don't have this isssue.  It will disappear once the options dialog is
  closed completely though.
