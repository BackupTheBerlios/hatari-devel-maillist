On Sun, 18 Oct 2009, Eero Tamminen wrote:

> The "problem" I was trying to solve was needing to give the --avi-fps option
> in bootup case.  But I guess most people will start it from the GUI anyway
> as recording the bootup would just slow down things & waste disk...

Yes, most people will use the gui ; the --avi-fps will be used in very 
rare case in a kind of "expert mode only".

>
> Maybe AviRecordDefaultFps should be zero by default and main.c complain
> if it's not set (with --avi-fps) when --avirecord is used?
>
> Or --avirecord option itself should require giving the FPS value instead of
> there being separate --avi-fps option?

Well --avi-fps is useful only when starting an immediate recording from 
the command line with --avirecord (from the gui, current refreshrate 
will be used, and that's the way to go).

when starting from the command line with --avirecord, either to record the 
boot sequence or to start an avi immediatly after restoring a memory 
snapshot, the default fps will be 50, which should cover nearly all use 
cases (since very few programs are really running in 60 Hz).

So defaulting to 0 to force the user to explicitly give a fps seems 
unnecessary to me, we can use a "best guess" with 50.

And in the rare case where you would really want to record at 60 Hz, then 
you need the optional --avi-fps parameter to do so.

So for me, both paramaters should be kept separated and giving the fps 
should not be mandatory (as most of the time you would give 50 which is 
the default value)
