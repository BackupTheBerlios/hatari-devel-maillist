Hi,

On Wednesday 09 December 2009, Konador, Cyprian wrote:
> I've tweaked a bit statusbar. I've add fast forward indicator,

I haven't added this information because doing ">" and ">>" with text
doesn't look so good and one can already see fast forward state from
the frameskip value if one is using auto-frameskip.

I would rather have them done with suitable icons/images.
At the same time LED drawing could be replaced with nicer LED images.

These would need images for different toolbar sizes and led states.  These
kind of images should be done only with couple of colors so that they aren't
completely awful even in 8-bit mode.


But if icons aren't feasible / Thomas doesn't like having them in Hatari,
then at least I would skip ">" and show only ">>" (when fast forward
is enabled).


> cpu type, cpu clock.

Where/when this information is needed?

The currently shown information is relevant for bug reports, but the shown 
machine type (ST/STE/TT/Falcon) already tells what you're proposing to show 
(unless cpu type & clock have been explicitly overridden by the user which
is pretty rare).

If there's no clear use for the extra information, I'd rather leave it out
as UI clutter.


If others think the information is userful, I think it would be clearer
if for 68000 it would just skip "000".  The information might also be
clearer if separated with "/":
	8Mhz/4MB ST, TOS v1.02
	32Mhz/030/14MB TT, EmuTOS
	16Mhz/040/8MB Falcon, EmuTOS

What do you think?


> LED are a bit smaller now.

They were wide to look more like (Atari) leds. :-)


> In additional, statusbar is visible  
> only in windowed mode, in fullscreen we can see overlay led. This is more
> pleasure, especially during watching demos/games.

You can switch statusbar to led from command line and from the options GUI.

Currently led drawing has more overhead than statusbar, that's why I don't
like it being hardcoded to be mode dependent.

If you want that kind of a change, you need to make it to be configurable
separately for fullscreen and windowed mode + allow changing them
from the options GUI.  The main issue is where in the GUI you can fit
the extra option(s)...

Current command line options can still set both at the same.
