Hi,

> I didn't know about the 68040 trick.  It indeed works.  Why that works with
> HD Driver instead of the 020+FPU that's enabled by Hatari Falcon mode
> normally?

It's because the Afterburner040 has _MCH set to 0x0003, which indicates
that it is a Falcon, but indicates that it has a 68020 CPU, even though
it has a 68040 CPU.
Unfortunately there are some combinations of cookies and processors used
by some hardware extensions that do not reflect the real processors and
hardware these hardware extensions are running with. This is why
HDDRIVER has to assume that certain combinations of cookies and
processors mean a certian hardware extension. In the case of Hatari this
does not work for the Falcon. On the other hand there never was a Falcon
with 68020 and Hatari does not offer an 68030 option, so the 68040 is
the best choice of processor for the Falcon when running Hatari.
I might be able to change the code in HDDRIVER so that it does not crash
anymore in this case, but it's a bit risky as I don't want to risk the
compatibility with numerous haredware add-ons for real Atari machines.
