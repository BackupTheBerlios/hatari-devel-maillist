Btw. What I was proposing earlier was moving the buttons to the left side
of the dialog.  When one is using scrollbar to navigate, the buttons are now
closer to that. If buttons would be on the left, then they would be closer
to mouse pointer when selecting files.


One more possibility could be to remove "Choose a file" text and put
the "Folder:" and the   | .. | ~ | / | buttons on the same line.

The "Choose a file" text is anyway misleading because the same dialog
is used also to select default floppy image directory and the GEMDOS
emulation directory.

If that text is left to the dialog, at least it should be changed according
to some flag that tells whether a file or folder is being selected.  When
selecting a folder, the dialog shouldn't show or update the file field.
