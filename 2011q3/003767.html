On Mon, 18 Jul 2011, Nicolas Pomarède wrote:

> So, solution is to use a lower value for fps_scale in the avi header.
>
> In avi_record.c, can you try changing lines 966/967 :
>       AviParams.Fps = Fps;
>       AviParams.Fps_scale = Fps_scale;
> with :
>       AviParams.Fps = Fps >> 8;
>       AviParams.Fps_scale = Fps_scale >> 8;
>
> This way fps_scale should be 65536 and you should not get warnings / errors. 
> If it doesn't work, try higher values 9, 10, ...
>
> If it works, I will update the sources accordingly.


Hi!

Yep this works :)

ffmpeg show the warning, but not fatal error, and the resulting file play 
in Quicktime Player.
