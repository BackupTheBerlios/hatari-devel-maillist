I just updated the video IO regs to return $OO in some cases instead of 
$ff, based on the dumps sent by Anders and Matthias for STE and Falcon.

I updated TT mode too, as it's quite likely it behaves this way too, but 
we would need the dump of a real TT to be sure.

This way, Awesome menu 16 correctly restores resolution in STF or STE 
mode when going back to the GEM.
