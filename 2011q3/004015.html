New cpu's integration is far from being complete (it will be complete 
when it can also replace the old cpu and run all STF/STE games/demos 
without any timings problem).
I don't think the 68030 emulation should slow down emulation that much, 
the cpu core structure is rather similar to the one in UAE, it just 
handles more opcodes and also different level of emulation (so I think 
WinUAE core should give similare results to UAE core in STF/STE mode)

But I think laurent also used some specific/more accurate dsp code when 
compiling with the winuae core, so this might not affect only cpu speed, 
but also other component.

In the meantime, until everything is fixed, it's quite possible that 
some demos that don't require the extra level of accuracy from WinUae 
core (68030 cycle counting) will run slower than with old UAE core.

All of this is still work in progress.
