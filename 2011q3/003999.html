Hello

I see some commit logs are a little too long on the 1st line (hello 
Laurent :) )

In order to keep the log messages concise and easier to read when doing 
a new release, plese try to respect the recommend mercurial guidelines :

  - 1st line of the changelog should be a summary of the change, it 
should not wrap on several lines and should be around 80-100 characters 
to fit on most screen (see http://hg.berlios.de/repos/hatari/summary, 
it's better if the summary of the change doesn't create several lines on 
screen).

  - other lines can be used for a more detailled description of the 
changes. Put as many lines as you like here.


Nothing serious, it will just give a nicer formatting of the changelog.
