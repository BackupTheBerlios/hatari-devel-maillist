Hi,

After looking into bug about (disk) speed issues with Kronos
benchmark application, I run GEMBench v4.03 CPU benchmarks on
ST/STE/TT/Falcon.

Results were quite surprising.  While ST & STE results
corresponded to the GEMBench reference values, TT and
Falcon values were off a lot for integer division and
float maths.

See the attached screenshots.  The "Reference" box tells
on what machine setup the test was run.

I guess this could explain why some things need to be run
at 32Mhz with Falcon emulation.

This was with the old COU core.  How much the speed numbers
are off with the WinAUE CPU core?


	- Eero
