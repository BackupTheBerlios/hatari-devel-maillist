Thanks for the test, indeed only bits 0 and 1 are kept from the values 
you wrote, which is what I expected (Hatari already handles it this way).

So, any value written in bits 2-7 is lost ; when reading on STF, bits 
2-7 are set to 1 (most of the time, there can be other variants) and on 
STE bits 2-7 are set to 0.
