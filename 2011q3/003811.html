Phheww, wrong alert :) The problem is due to this specific demo, not to 
Hatari.

This is what happen at boot :
  - ff820a is set to 0 by default in the shifter, so any ST will always 
be 60 Hz when switched on

  - if you use a PAL tos, it will switch to 50 Hz
  - if you have an NTSC tos, it won't touch freq reg anymore, so the 
display stays at 60 Hz.

Now, Froggies over the fence expects to be started from the boot disk, 
which is the case, so resolution will already be 0, but freq could be 0 
or 2, and the demo doesn't force it to 2 to be sure it goes to 50 Hz.

And as no part of the demo is removing border until you reach the menu, 
the ST stays in 60 Hz and the demo goes wrong (it would be the same on a 
real ST)

So, yes, it's better to always boot with a PAL/european TOS to watch 
demos/games.
