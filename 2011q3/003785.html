On tiistai 19 heinäkuu 2011, Nicolas Pomarède wrote:
> Le 19/07/2011 11:18, Vincent Rivière a écrit :
> > However, I notice that inside both the binary and source archives,
> > there is a file named tos.img, which is actually an EmuTOS binary.
> > 
> > Providing EmuTOS with Hatari is great for everyone, it allows Hatari to
> > be ready to use, and popularize EmuTOS.
> > 
> > But EmuTOS is GPL, and you know the GPL rules: you can redistribute any
> > binary of GPL software (even modified) as long as you also redistribute
> > the exact sources you used to build it. I don't see the EmuTOS sources
> > in your download area, so it looks like Hatari is currently violating
> > EmuTOS' GPL.

We're using option c) from the EmuTOS GPL license:

---
  3. You may copy and distribute the Program (or a work based on it,
under Section 2) in object code or executable form under the terms of
Sections 1 and 2 above provided that you also do one of the following:
...
    c) Accompany it with the information you received as to the offer
    to distribute corresponding source code.  (This alternative is
    allowed only for noncommercial distribution and only if you
    received the program in object code or executable form with such
    an offer, in accord with Subsection b above.)
---

EmuTOS binary packages have note about where the source is
available only in the doc/announce.txt file:
---
AVAILABILITY

EmuTOS has its home at sourceforge:

    http://sourceforge.net/projects/emutos

A ready made EmuTOS image or the source can be downloaded from:

    http://sourceforge.net/project/showfiles.php?group_id=36560

It is always available in source form from our CVS server at:

    http://sourceforge.net/cvs/?group_id=36560
---

> > "make 512". This is very easy and I hope this will be OK for you.
> > 
> > Thanks in advance for considering this issue.
> 
> Are you sure the GPL requires to put the emutos sources *inside* Hatari
> released archives ?

No.  Vincent suggested adding separate source tarball to the dowload area.


>  From what I understood, the GPL requires to publish the sources you
> modified (which is not our case, we compiled emutos without changing
> anything in the source) and to include some informations about the
> project, which is the case in doc/emutos.txt, with a direct link to
> http://sourceforge.net/projects/emutos
>
> Since we didn't modify the source, I don't think the GPL implies we
> distribute a copy of the source, giving an url to the official emutos
> site should be enough to comply.

I think most appropriate would be to include also a direct link to
correct source code package version, not just the project page, e.g:
	http://sourceforge.net/projects/emutos/files/emutos/0.8.5/

and state that we're distributing unmodified EmuTOS binaries.
