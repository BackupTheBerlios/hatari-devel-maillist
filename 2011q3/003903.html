I now read through the memory test of EmuTOS, and I think I
understood the problem. The memory test of EmuTOS relies on some
bank mirroring of the real hardware, which is not emulated in Hatari
yet (and probable in a couple of other emulators). The problem only
occurs for memory configurations < 4 MB which require two memory banks,
i.e. 512 kB and 1 MB.
Since this might be a problem on multiple emulators, I've added another
sanity memory check there, so that the memory init now also works when
the emulator does not support the bank mirroring.
