On keskiviikko 17 elokuu 2011, Vincent Rivière wrote:
> Vincent Rivière wrote:
> > I have just fixed that in the EmuTOS CVS.
> 
> I have just noticed one strange thing.
> 
> EmuTOS detects the Microwire interface by reading a byte at 0xffff8922.
> This does not cause a bus error on Hatari with Falcon mode. Is this the
> expected behavior? I understood that the Falcon has no Microwire
> interface and LMC1992.

If memory isn't present, trying to access that address area of course
creates an exception.

However, whether some HW is present and whether the corresponding
control register address in RAM generates an exception are different
things.

Thomas has a test program and results from runs on different Atari
machines in different modes in the Hatari's tests/buserror/results/
-directory.

According to that data, in none of the modes (byte/word access,
STE compatible mode on/off) address 0xffff8922 will give a bus
error.
