Hi,

I've retested rainbow 2 multimedia.

With old CPU, it crashs with bombs (MMU problem)
With new cpu, it loops forever in a sync test (there's a MMU emulation 
for 68030 in the new core)


The infinite loop is related to Videl emu : they test $FFFF82A0 (word) : 
VFC - Vertical Frequency Counter for synchronisation :

First they test if VFC == 0, then if VFC != 0

Like this, they're sure to start with the first displayed line.

I'll fiw this with the videl emulation soon

Should we mark this in the compatibility list ?
