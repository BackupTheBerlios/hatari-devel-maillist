> And start working on the new stuff:

I thought a little more about this one, see the thread about this.

> - make WinAUE the default CPU core

UAE please :)

As for making it the default CPU core, we're faaaar from that point, no 
work has been done on it recently and I'm not sure people realise the 
workload it means and that it will impact lots of important place 
(video, mfp, ...).
Honestly, given the pace at which it evolves, I don't see a full working 
replacement before one year or more.

> - start fixing WinAUE debugging support (add exception debugging,
>    fix what it does on reset etc)

Let's not add complexity to a part that does not fully replace the 
current cpu core yet.
For now, people will have to live without the debugger features added to 
the current cpu core not being present in winuae's core.
First step is really to make winuae's core works, and then we will 
backport the modifications from current cpu.

The more we add code to Winuae's cpu, the more complex it will be to fix 
real emulation issues (and there're a lot of them to be able to 
integrate in STF/STE mode).
