Hello,

Videl emulation is really basic for now.

Videl is based on precise Cycles to run.
For now, 68030 cycles emulation is not perfectly emulated (from my point of view).

So, for now, I've decided to merge WinUae's cpu core with Hatari (68030 emulation is better).

After this, I'd like to rework the DSP emulation in precise CPU emulation mode (I would get real frequencies for crossbar (DMA sound and DSP <-> Host transfers), Host port (port B)  and port C.

Later will comes videl's turn.

All of this should end with a more compatible Hatari emulation in Falcon mode.
