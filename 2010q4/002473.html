> As I'm both Amiga user and CT60 coder, I add my 2 cents here.
>
> 1. No 060 demo I know uses any of 060 instructions, move16 is the max 
> you can get from non-030 code and even this is rare (it's more 
> effective to use movem for such tasks, i.e. 060 coders ignore move16). 
> This applies to both Amiga and Atari. Try to run any 060-based 
> production in Hatari -- if you increase RAM to 64 MB, you'll run it 
> fine. In fact, many 060 coders use current Hatari for their 
> development ;-)
>
> 2. Maybe it's just me but I was actually never able to run an 060 demo 
> in 68060 mode in WinUAE ;-) Every 060 demo I ran was with 020+FPU 
> configuration.
>
> So I guess the MMU is the only benefit you get here.
