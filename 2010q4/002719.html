Hi,

there is a problem with hatari which is probably not completely new, but
did not exist with hatari based on the sources from 2010-09-26.
With the current sources something is wrong when using NVDI. I can only
give a general description of the problem, maybe somebody knows about a
relevant change.
When using the TARKUS software (a tool for printing labels) the print
preview (as well as the actual printout) do not correctly recognize the
paper size (A4) anymore. The output is clipped, so that only the upper
left corner is visible. My guess is that NVDI relies on data returned by
the actual VDI and these data are probably not correct anymore.
When using a copy of hatari compile in September (without changing any
settings) everything is fine.

Best regards
