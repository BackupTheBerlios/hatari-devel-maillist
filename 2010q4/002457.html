> So, I'm afraid the whole disasm should be written from scratch as GPL,

Or ask the auther to re- or dual-license his code.


Or have in Hatari code support both for UAE and the MAME disassembler
and default for the UAE one.  GPL licence is incompatible only in case of
binary distribution. But there's still the additional problem of the source
distribution by commercial Linux distros (e.g. for price of storage media),
which could be solved only by providing the non-free MAME disassembler
code separately from the main Hatari source package.

I'd rather hope the disassembler sources get re-/dual-licensed.
(e.g. with the exception that such an extra licence applies only when
the source is used together with Hatari emulator.)
