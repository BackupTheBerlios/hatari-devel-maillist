Hello

when using 224 bytes overscan on STE + horizontal scrolling, the last 16 
pixels were not correctly displayed, as there was only a case for 230 
bytes overscan + hscroll in Hatari.

I changed the conversion routine to a more generic one that handles any 
line's length ; the last pixels will now be correct for 160 bytes lines, 
230 bytes, 224 bytes or any other length.

I checked with a few demos using different overscans, hopefully this 
should be good ; let me know if you see a regression.

This fix demos from DHS (Cernit and More Or Less Zero) as well as Save 
The Earth by Defence Force.


Nicolas
