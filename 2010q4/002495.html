Ah, right, I somehow had though you meant the function that gives file names
suitable for tmpfiles it returns an opened file handle.  As tmpfile() opens
& returns the file handle, it can already unlink the file.


Anyway, I got it now implemented and commited the support & updated
documentation to the repo.  Please everybody, test with different TOS
versions, programs and different Hatari arguments.  Also whether this works
on OSX and Windows in addition to Linux.


There's also the script for registering Hatari as TOS program runner on
Linux boot using the binfmt_misc feature.

On normal Desktop I would recommend using the Desktop's features for
this though, for example in KDE3 you can do that from:
	Control Center -> KDE Components -> File associations
Then just add file patterns for *.tos, *.prg, *.ttp and hatari as
handler for those.


The current code has couple of optional debugging features (desktop file
can be set to be there for more than only first access after boot, writing
the INF file to current dir for examining).  I'll remove those when this
feature is found to be working fine.
