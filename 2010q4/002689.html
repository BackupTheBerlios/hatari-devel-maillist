Hello,

I wanted to make a little break with newcpu, so I returned to DSP code.

I've uploaded some big changes :

First in dsp_disasm :

I've hardcoded every parallel instructions in a string table.
No more decoding for these instructions.
This allowed to removed nearly 1/3 of the disasm.c code.



Second in dsp_cpu.c (2 major changes) :

- I've hardcoded all 256 parallel instructions (this result in more 
lines of code)

- I've removed dsp_pm_read and dsp_pm_write functions + all functions 
relative to parallel moves.
   I've coded these moves differently in one call.
   As a bonus, I think code for parallel moves is easier to read like 
this, as it is implemented directly in each dsp_pm_xx function.


All of these changes should result in a little more dsp speed.
Thomas, can you test this and tell me if this speeds up your falcon ?

I've tested many dsp programs and didn't noticed any regression.
